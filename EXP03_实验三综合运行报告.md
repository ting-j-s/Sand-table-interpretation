### 3.2 拓扑对成本路由的影响

```
Small-World (实验二, 高聚类 C=0.177):
  → 适合分布式检测任务
  → 每个Agent与邻居紧密通信
  → 消息需要多跳才能到达Hub
  → 成本: 更多节点参与决策, V3.2/Pro调用更多

Scale-Free (实验三, 高枢纽分化):
  → 适合集中式调度任务
  → Hub节点承担路由决策
  → 消息直达Hub, 少量节点调用Pro
  → 成本: 仅Hub节点使用Pro, 其余→Flash
  → 预期成本节约: +15-20%

拓扑选择建议:
  检测场景 → Small-World (高检测率, 成本较高)
  路由场景 → Scale-Free (低成本, Hub集中决策)
  混合场景 → Modular (功能分区, 平衡成本与检测)
```

## 8. 关键发现

### 8.1 D06路由优化的核心发现

当前Fast Keyword分类器因任务类型定义偏差导致Pro占比过高(29.3% vs 目标5%)。优化分类器定义可将成本降至~$77(50K任务)，实现vs All-Pro的~60%成本节约。预算网关机制完整但未被触发(31%使用率)。

### 8.2 拓扑-成本关联

| 拓扑 | 聚类系数 | 适合场景 | 成本特征 |
|------|---------|---------|---------|
| small-world | 0.177 (高) | 分布式检测 | 多节点参与, 成本较高 |
| scale-free | 0.089 (低) | 集中式路由 | Hub集中决策, 成本最低 |
| modular | 待测试 | 功能分区 | 按模块分配, 成本可控 |

### 8.3 沙盘收敛的鲁棒性

三次独立实验均收敛于**9轮**。无论攻击链如何变化(实时注入/数据投毒/成本优化)，D03沙盘的收敛行为保持一致，证明模型具有良好的可重复性。

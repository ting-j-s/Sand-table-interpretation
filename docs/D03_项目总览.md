# D03 沙盘演绎 — 项目总览

> 真实 API 驱动的网络靶场沙盘推演系统 | ~8,500 行 Python | 24 个模块

## 一、项目定位

构建**真实 LLM API + 真实威胁情报**驱动的四方沙盘推演系统。公司/用户/蓝队/红队四方智能体均由真实大模型调用驱动决策，威胁情报从 VirusTotal/AbuseIPDB/Shodan 等 API 实时拉取，不做数学公式模拟。

```
外部能力 ────────────────────────────────────────────── 内部系统
          │                                              │
  DeepSeek/OpenAI/      →  real_llm.py                  │
  Anthropic/智谱/Moonshot   D03_real_llm_bridge.py       │
          │                    ↓                         │
  VirusTotal/AbuseIPDB/ →  D03_real_threat_intel.py      │
  Shodan/AlienVault OTX     D03_ti_deepseek_engine.py    │
          │                    ↓                         │
          │              D03_real_agents.py  ←─── D03_personality_kb.py
          │              D03_real_sandbox.py              │
          │                    ↓                         │
          │              integration/ (编排器+评估)      │
          │                    ↓                         │
          │              D03_security_logger*.py          │
          │              D03_log_*.py (ML训练)           │
          │                                              │
          └── 真实API决策 ──→ 四方沙盘推演 → 日志 → 检测评估
```

## 三、模块详解

### 3.1 API 与 LLM 层

**`D03_real_api_config.py`** — 多 Provider API 密钥管理
- `APICredential`: 单个凭据（provider / key / base_url / rate_limit）
- `RealAPIConfig`: 全局配置，从环境变量读取所有 API 密钥
- 支持 Provider: DeepSeek / OpenAI / Anthropic / 智谱 / Moonshot
- 连接池、速率限制、用量追踪

**`real_llm.py`** — LLM 调用核心
- `call_llm(model, system_prompt, user_prompt, ...)`: 统一 API 调用接口
- 返回 token 数、成本（USD）、延迟
- Model alias: `FLASH` → deepseek-v4-flash, `PRO` → deepseek-v4-pro
- 辅助函数: `classify_complexity()`, `generate_poison_sample()`, `red_team_attack_plan()`, `blue_team_defense_plan()`, `evaluate_output_harmfulness()`

**`D03_real_llm_bridge.py`** — LLM 桥接层
- `RealLLMBridge`: 通用 LLM 调用封装（chat_json 结构化输出）
- `SOCLLMBridge(RealLLMBridge)`: SOC 告警研判通道 — 输入告警信息 → 输出优先级、处置建议、是否升级
- `RedLLMBridge(RealLLMBridge)`: 红队攻击规划通道 — 输入目标画像 → 输出攻击阶段、MITRE 技法、成功率预估

### 3.2 威胁情报层

**`D03_real_threat_intel.py`** — 多源威胁情报聚合
- `ThreatIntelHub`: 基类，统一缓存与 API 调用模式
- `VirusTotalClient`: 文件/URL/IP 查询，恶意检出引擎统计
- `AbuseIPDBClient`: IP 滥用置信度评分
- `ShodanClient`: 开放端口、服务、已知漏洞
- `AlienVaultOTXClient`: 威胁脉冲/IOC 关联
- `URLScanClient`: URL 安全扫描
- `ThreatIntelAggregator`: 聚合全部源 → `full_ip_intel(ip)` 返回综合威胁画像

**`D03_ti_deepseek_engine.py`** — LLM 增强威胁分析
- `DeepSeekTIEngine`: 将原始情报数据交给 DeepSeek 做语义分析
- 功能: 威胁报告生成、IOC 关联推理、攻击者画像

### 3.3 多智能体层

**`D03_real_agents.py`** — 真实 LLM 驱动的多智能体
- `AgentMemory`: 智能体记忆（观测/行动/反馈三元组）
- `RealRedAgent`: 红队攻击智能体 → LLM 决定攻击向量
- `RealBlueAgent`: 蓝队防御智能体 → LLM 研判 + 响应
- `RealGreenAgent`: 绿队建设智能体 → LLM 优化安全架构
- `RealPurpleAgent`: 紫队评估智能体 → LLM 综合评估
- `RealMultiAgentOrchestrator`: 多智能体编排（组队/轮次/战报）

**`D03_personality_kb.py`** — 人格心理学知识库
- `HEXACOProfile`: 六因素人格（诚实-谦逊/情绪性/外向性/宜人性/尽责性/开放性）
- `DarkTetradProfile`: 暗黑四特质（自恋/马基雅维利/精神病态/施虐）
- `CognitiveBiasProfile`: 12 种认知偏差
- `MBTIStyle`: Myers-Briggs 16 型
- `FullPersonalityProfile`: 完整人格画像
- `PersonalityKnowledgeBase`: 知识库查询接口

### 3.4 沙盘层

**`D03_real_sandbox.py`** — 四方沙盘核心
- `RealCompanyAgent`: 公司层 — LLM 驱动的预算分配与安全策略
- `RealUserAgent`: 用户层 — 人格驱动的行为生成
- `RealBlueAgent`: 蓝队 — 检测/响应/MTD
- `RealRedAgent`: 红队 — 攻击链路生成
- `RealOrchestrator`: 四方编排器 — 每轮推演 + 评分
- `APICostTracker`: API 调用成本追踪

**`D03_real_runner.py`** — 统一运行入口
- `D03RealRunner`: CLI 参数解析 → 模块组装 → 运行 → 报告
- 串联: API 配置 → 威胁情报 → LLM 桥接 → 多智能体 → 沙盘 → 编排器 → 报告

### 3.5 编排管道（integration/）

**`D03_沙盘编排器_基础版.py`** — 主编排器
- `SandboxOrchestrator`: 多轮红蓝对抗主循环
- 每轮: 红队生成攻击计划 → SOC 研判 → CISO 策略优化 → 安全评分
- 支持 `use_real_api=True/False` 切换真实/模拟模式

**`D03_沙盘编排器扩展_红蓝对抗.py`** — 扩展红蓝对抗
- `RedTeamAgent` / `BlueTeamAgent`: 带能力值和经验增长的智能体
- `ScenarioEngine`: 多场景配置（金融/医疗/能源）
- `MITREScoringEngine`: 基于 MITRE ATT&CK 的评分
- 15 种攻击战术枚举

**`D03_评估引擎_逃逸成功率与成本.py`** — 沙箱评估
- `EscapeSuccessRate`: 15 种逃逸技术 × 5 种沙箱类型的 ESR 矩阵
- `DefenseEvaluator`: 10 层防御有效性评估
- `CostModel`: 攻击成本 / 防御成本 / ROI
- `SandboxBenchmark`: 5 沙箱全维度对比

### 3.6 日志与检测

**`D03_security_logger.py`** — 企业安全日志生成
- `EnterpriseSecurityLogger`: 从沙盘状态生成 9 种格式日志
- Palo Alto 防火墙 / Windows EventLog (4624/4625) / BIND DNS / Suricata IDS / Sysmon 端点 / Squid 代理 / Proofpoint 邮件 / Splunk SIEM / NetFlow

**`D03_security_logger_v2.py`** — 增强版日志
- 改进: IP 多样性 / EventID 变化 / 端口分布 / 严重度曲线 / 日志长度方差

**`D03_log_generator.py`** — 批量日志生成
- `CompanyLogGenerator`: 按轮次批量生成公司所有日志，支持多轮迭代

**`D03_log_ml_trainer.py`** — ML 训练
- `LogFeatureExtractor`: 从日志提取 40 维特征
- `SecurityModelTrainer`: RF/GB/MLP 训练 + StratifiedKFold 验证

### 3.7 分析与存储

**`D03_mitre_mapper.py`** — MITRE ATT&CK 映射
- `MITREMapper`: 41 技术 × 14 战术映射，覆盖率评分，缺口分析

**`D03_industry_benchmark.py`** — 业内标准对比
- 9 种模型: 监督（LogReg/RF/GB/MLP/Ensemble）+ 无监督（AE/IF）+ 半监督

**`D03_memory_store.py`** — 持久记忆
- `MemoryStore`: Working / Session / Persistent 三层记忆
- `D03MemoryManager`: 记忆生命周期管理

**`D03_framework_adapter.py`** — 框架适配
- Agent 定义适配器，桥接不同智能体框架

### 3.8 控制面板

**`d06_control.py`** — D06 CLI 统一控制
- `cmd_status()` — 查看系统状态
- `cmd_run(exp_num)` — 运行指定实验
- `cmd_bench()` — 跑基准测试
- `cmd_report()` — 生成报告
- `cmd_api()` — API 连接检查
- `cmd_defend()` / `cmd_attack()` — 攻防操作
- `cmd_adversarial()` — 对抗训练
- `cmd_pipeline()` — 端到端管线

## 四、全流程验证

`scripts/_test_final.py` 提供 6 步端到端验证：

```
1/6  API & Model Check       → LLM 连通性，Flash/Pro ping
2/6  Threat Intelligence     → 4 个 IP 多维情报查询
3/6  SOC Triage              → 真实 LLM 研判告警
4/6  Red Team Plans          → 多行业攻击计划生成
5/6  Sandbox Orchestrator    → 6 轮真实 API 沙盘推演 + 成本追踪
6/6  Multi-Agent Red/Blue    → 4 轮多智能体红蓝对抗
```

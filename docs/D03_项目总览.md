# D03 沙盘演绎 — 项目总览

> 版本 v9.0 | 2026-05-21 | 四方网络靶场沙盘 + ML训练 + 安全日志生成 + 沙箱逃逸评估 + LLM智能体 + Claude Code架构优化

## 一、项目概述

构建四方网络靶场沙盘，模拟真实企业安全运营全景。四方分别为：**公司管理层**(战略决策/资源配置)、**用户员工层**(OCEAN人格数字孪生)、**蓝方维护层**(SOC监控/防御/响应)、**红方攻击层**(复合攻击)。四方在统一Agent框架(AIgentDefinition/Tool/Registry/Permission继承链/三层记忆)和强化学习驱动的沙盘总控下持续博弈。

## 架构参考: Claude Code 源码模式迁移

基于 Claude Code ~512K行TypeScript源码分析，迁移6个核心模式:
- **AgentDefinition联合类型** (loadAgentsDir.ts): BuiltIn/Custom/Plugin三层Agent定义
- **buildTool()工厂** (Tool.ts): 自描述工具+权限检查+isConcurrencySafe标记
- **权限继承链** (runAgent.ts:414-497): 子Agent不可升级权限(parent AUTO/BYPASS始终优先)
- **ToolSearch延迟加载** (ToolSearch.ts): 工具schema按需加载,节省上下文token
- **三层记忆** (Chapter 9): Working/Session/Persistent + 不可派生原则
- **AsyncGenerator引擎** (runAgent.ts:748-806): yield流式执行+finally资源清理

## 五-B、沙箱逃逸评估体系 (EXP10-EXP13)

### EXP10: 大规模沙箱逃逸基准
- 5,000 逃逸样本 + 5,000 正常样本
- 15 种逃逸技术 (container_breakout ~ resource_exhaustion)
- 5 种沙箱类型 (Docker/K8s/Firecracker/gVisor/Kata)
- 整体ESR: 8.33%, 检测率: 91.67%, 准确率: 76.99%
- 10层防御: syscall分析/能力审计/命名空间完整性/seccomp验证...

### EXP11: 对抗强化循环
- 3轮攻防对抗, 每轮4,000操作
- 论文知识库驱动: 30+篇攻防论文
- 自适应攻击聚焦: 每轮根据上一轮弱点调整
- 最终F1: 0.9773 (从0.9656提升+0.0117)
- 容器逃逸ESR从3.76%降至1.79%(-52.4%)

### EXP12: 集成红队沙箱逃逸
- 四方沙盘(MITRE ATT&CK) + 沙箱逃逸评估桥接
- 15技术 → MITRE ATT&CK映射(T1611/T1068/T1562.001...)
- 红队智能体: 技术组合 + 技能学习(+0.02/成功未检测逃逸)
- 蓝队SOC: ML集成(RF/GB/LogReg) + GNN图异常检测
- 100轮模拟: 999次逃逸尝试, ESR=0.1562

### EXP13: 端到端生产管线验证
- 6阶段管线: 沙盘仿真→日志生成→ML检测→GNN检测→逃逸评估→性能采集
- 5沙箱对比矩阵: Docker/K8s/Firecracker/gVisor/Kata
- Docker ESR=0.203 (最高) vs Kata ESR=0.013 (最低, 93.6%更低)
- ROC AUC=0.837, PR AvgPrecision=0.887
- 成本收益: 攻击成本/防御成本/ROI/违规成本节省

## 六、关键修复记录

| 问题 | 原因 | 修复 |
|------|------|------|
| 蓝队检测恒定0.920 | 硬编码cap `min(0.92,...)` | 改为0.95 + 事件驱动logistic增长 |
| 红队技能恒定0.633 | `successful_attacks > failed_attacks` 新智能体总为False | 改为所有智能体每轮自动学习 |
| 5分钟检查值不变 | 确定性公式 `gap * rate` | 改为事件驱动批处理(每500轮) + 随机扰动 |
| 攻击事件溢出 | 无清理机制 | 每1000轮裁剪至25,000条 |
| ML F1虚假1.000 | 简单RF只预测多数类 | 替换为sklearn + StratifiedKFold + class_weight |
| 数据泄漏F1=1.000 | 标签特征在训练集中 | 移除4个泄漏特征 + per-fold SMOTE |
| 日志99.5%攻击标签 | 累积exfil永远>阈值 | 改用逐轮增量delta标签 |

```
安全日志 → 关系图构建 → 5轮PageRank消息传递 → 节点特征 → 图级聚合 → 攻击检测

边权重: DENY=2.0, 可疑端口=1.5, 认证失败=2.0, 恶意DNS=3.0, CRITICAL IDS=3.0, RemoteThread=3.0

图级特征: [Nodes, Edges, Density, Suspicious%, Max/Avg Risk, Max/Avg Degree, Centrality, RiskStd, HighRisk%]
```
